The Solar Estimate module calculates annual and monthly solar energy production for a photovoltaic system at any location worldwide. It uses NREL's PVWatts V8 API under the hood, so you get industry-standard irradiance data without needing a separate PVWatts account. Results update in real time as you adjust parameters, and you can export the full model to XLSX.

## What it produces

Running the Solar Estimate module generates an interactive model widget in your project chat showing:

- **Annual AC energy output** (kWh/yr)
- **Capacity factor** (%)
- **Annual solar radiation** (kWh/m²/day)
- **Monthly production breakdown** — AC energy, DC energy, irradiance, and plane-of-array data for all 12 months
- A **data confidence label** based on the weather station used

The model widget stays live in your chat — you can adjust any input and recalculate without re-running the module.

## How to use it

<Steps>
  <Step title="Open your project and select Solar Estimate">
    In your project chat, use the tool picker or type a request like "Run a solar estimate for this project." The Solar Estimate workspace opens.
  </Step>
  <Step title="Set your location">
    Enter an address or place name. The module geocodes it to latitude/longitude automatically. You can also enter coordinates directly.
  </Step>
  <Step title="Review and adjust parameters">
    The AI pre-fills system parameters from your project context. Review each input and confirm or override values as needed.
  </Step>
  <Step title="Recalculate">
    Click **Recalculate** to run the PVWatts API and see updated results instantly.
  </Step>
  <Step title="Export to XLSX">
    Click **Export** to download a formatted Excel workbook with a Summary sheet (headline results + monthly table) and an Inputs sheet.
  </Step>
</Steps>

## Input parameters

The following parameters control the solar estimate calculation:

<ParamField path="system_capacity" type="number" required>
  DC system size in kilowatts (kW). This is the nameplate capacity of the PV array.
</ParamField>

<ParamField path="address" type="string">
  Street address or place name used to determine the site location and weather data. Converted to lat/lon automatically.
</ParamField>

<ParamField path="lat" type="number">
  Site latitude in decimal degrees. Set automatically from address geocoding, or enter directly.
</ParamField>

<ParamField path="lon" type="number">
  Site longitude in decimal degrees. Set automatically from address geocoding, or enter directly.
</ParamField>

<ParamField path="tilt" type="number">
  Panel tilt angle in degrees from horizontal. Typical values: 0–30° for equatorial locations.
</ParamField>

<ParamField path="azimuth" type="number">
  Panel azimuth (compass direction the panels face) in degrees. 180° = south-facing (northern hemisphere default).
</ParamField>

<ParamField path="array_type" type="integer">
  Mounting configuration. Options:
  - `0` — Fixed (open rack)
  - `1` — Fixed (roof mount)
  - `2` — 1-axis tracking
  - `3` — 1-axis backtracking
  - `4` — 2-axis tracking
</ParamField>

<ParamField path="module_type" type="integer">
  PV module technology. Options:
  - `0` — Standard (crystalline silicon)
  - `1` — Premium (high-efficiency silicon)
  - `2` — Thin film
</ParamField>

<ParamField path="system_losses" type="number">
  Total system loss percentage (%). Accounts for wiring, soiling, shading, and other losses. Default is typically 14%.
</ParamField>

## Output fields

| Field | Description |
|---|---|
| `ac_annual` | Annual AC energy production (kWh/yr) |
| `capacity_factor` | System capacity factor (%) |
| `solrad_annual` | Annual average solar radiation (kWh/m²/day) |
| `ac_monthly` | Array of 12 monthly AC energy values (kWh) |
| `dc_monthly` | Array of 12 monthly DC energy values (kWh) |
| `solrad_monthly` | Monthly solar radiation (kWh/m²/day) |
| `poa_monthly` | Monthly plane-of-array irradiance (kWh/m²) |
| `quality_label` | Weather data confidence level |

## Export format

The XLSX export contains two sheets:

- **Summary** — headline results and a full monthly production table (AC, DC, solar radiation, POA irradiance)
- **Inputs** — all input parameters with their values, units, confirmation status (confirmed / assumed), and notes

<Note>
  Input rows highlighted green are values you confirmed directly. Yellow rows are assumed defaults. Review assumed values carefully before using the export in financial models.
</Note>

## Tips

<Tip>
  If your project already has a location set in the intake stage, the Solar Estimate module pre-fills the address automatically.
</Tip>

<Tip>
  Use 1-axis tracking (`array_type: 2`) for utility-scale ground-mount systems in high-irradiance locations — it can increase yield by 20–30% over fixed-tilt.
</Tip>
