The LCOE (Levelized Cost of Electricity) model calculates how much each kilowatt-hour of electricity produced by your project costs over its lifetime, expressed in your chosen currency per kWh. It's the standard metric for comparing energy technologies and evaluating whether a project is economically viable. Nitrogen AI reads your project documents and conversation to extract inputs automatically, fills gaps with technology-appropriate defaults, and explains every assumption it makes.

## What it calculates

<CardGroup cols={2}>
  <Card title="LCOE ($/kWh)" icon="bolt">
    The core output: total lifecycle cost divided by total discounted energy production. Expressed in your project currency per kWh.
  </Card>
  <Card title="Cost breakdown" icon="chart-pie">
    How much of the LCOE comes from CAPEX, O&M, fuel, and replacement costs — shown as both absolute values and percentage shares.
  </Card>
  <Card title="NPV of costs" icon="dollar-sign">
    The net present value of all project costs over the lifetime, discounted at your WACC.
  </Card>
  <Card title="Sensitivity analysis" icon="chart-bar">
    How the LCOE changes as key inputs vary ±20%, so you can identify which assumptions matter most.
  </Card>
</CardGroup>

## How it works

<Steps>
  <Step title="Nitrogen AI extracts inputs">
    When you mention LCOE or project economics in the chat, the module reads your conversation and any uploaded documents to pull out numeric values. It understands unit conversions (MW → kW, decimal capacity factors, etc.) and knows which technology defaults to apply if certain values are missing.
  </Step>
  <Step title="Review the inputs table">
    An inputs table appears in the chat showing every parameter with its value, unit, source (extracted from your documents or assumed), and a status indicator. Assumed values are highlighted so you know exactly where the model is filling in gaps.
  </Step>
  <Step title="Edit and recalculate">
    Change any value directly in the table. The LCOE recalculates instantly — no need to re-run the full module. You can also type changes in the chat: "change the WACC to 12%" or "set project life to 20 years".
  </Step>
  <Step title="Export the model">
    Download a formula-driven Excel workbook. Change any input on the Model sheet and every other sheet — Cash Flows, Cost Breakdown, Sensitivity — recalculates automatically.
  </Step>
</Steps>

## Inputs

### Required

<ParamField path="body.net_capacity_kw" type="number" required>
  Installed system capacity in kilowatts (DC nameplate). The model uses this with the capacity factor to calculate annual energy production.

  *Example: 500 (for a 500 kW solar plant)*
</ParamField>

<ParamField path="body.total_capex" type="number" required>
  Total capital expenditure in the project currency. Include all upfront costs: equipment, installation, civil works, and development costs.

  *Example: 750000 (USD)*
</ParamField>

<ParamField path="body.annual_opex" type="number" required>
  Annual operations and maintenance cost in the project currency per year.

  *Example: 15000 (USD/yr)*
</ParamField>

### Optional (filled with technology defaults if not provided)

<ParamField path="body.capacity_factor" type="number" default="Technology default (e.g. 0.18 for solar PV)">
  Expected capacity factor as a decimal between 0 and 1. Represents the fraction of maximum possible output the system actually generates.

  *Example: 0.18 (18% for a solar PV system)*
</ParamField>

<ParamField path="body.discount_rate" type="number" default="0.08">
  Weighted average cost of capital (WACC) as a decimal. Used to discount future costs and energy production to present value.

  *Example: 0.10 (10% WACC)*
</ParamField>

<ParamField path="body.project_life_years" type="number" default="25">
  Operational lifetime of the project in years. Technology defaults: solar PV 25 years, wind 20–25 years, mini-grid 15–20 years.

  *Example: 25*
</ParamField>

<ParamField path="body.annual_fuel_cost" type="number" default="0">
  Annual fuel cost (diesel, biomass, LPG, etc.) in the project currency per year. Set to 0 for fuel-free technologies like solar PV or wind.

  *Example: 50000 (for a diesel hybrid system)*
</ParamField>

<ParamField path="body.annual_replacement_cost" type="number" default="0">
  Annual cost of component replacements (e.g. battery packs) averaged over the project life.
</ParamField>

<ParamField path="body.degradation_rate" type="number" default="0.005">
  Annual decline in energy output as a decimal. Accounts for panel degradation, equipment aging, etc.

  *Example: 0.005 (0.5% per year, typical for solar PV)*
</ParamField>

<ParamField path="body.construction_years" type="number" default="0">
  Number of years before the system begins generating electricity. CAPEX is spread evenly across these years. Set to 0 for projects with immediate commissioning.
</ParamField>

<ParamField path="body.currency" type="string" default="USD">
  Currency code for all monetary values in the model and export.

  *Example: USD, EUR, KES, GHS*
</ParamField>

<ParamField path="body.technology_type" type="string">
  The energy technology type. Used to select appropriate defaults when inputs are missing.

  Options: `solar_pv`, `wind`, `battery`, `mini_grid`, `clean_cooking`, `other`
</ParamField>

## The exported Excel workbook

The XLSX export contains four sheets:

| Sheet | Contents |
|-------|----------|
| **Model** | Key inputs (editable), derived values, LCOE result, and cost breakdown percentages. Change any input here and the whole workbook recalculates. |
| **Cash Flows** | Year-by-year breakdown of CAPEX, O&M, fuel, replacement, energy production, and discounted values across the full project life. |
| **Inputs** | Complete provenance table: every parameter with its value, unit, source, status (confirmed vs. assumed), and notes. |
| **Sensitivity** | How the LCOE changes as each key parameter varies from its base value, with Δ from base shown as a percentage. |

<Tip>
The Model sheet is designed to be your working file. Share it with your finance team — they can adjust the WACC, capacity factor, or CAPEX and see how those changes flow through to the LCOE immediately.
</Tip>

## Estimate quality

Each calculation includes an estimate quality indicator: **high** (all key inputs confirmed by you), **moderate** (a few assumed defaults), or **low** (many assumed defaults). The Inputs sheet highlights assumed values in yellow so you can see exactly where the model is making guesses.

<Warning>
The LCOE model produces estimates based on the inputs you provide. Assumed default values are drawn from technology benchmarks and may not reflect your specific project conditions. Always validate critical inputs — especially capacity factor, CAPEX, and discount rate — against your project's actual data before making investment decisions.
</Warning>
