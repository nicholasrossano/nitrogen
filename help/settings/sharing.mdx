---

Project sharing controls who can see and work on an individual project. It's managed per-project from that project's **Share** panel, not from a single global settings screen.

## Roles

<CardGroup cols={3}>
  <Card title="Owner" icon="crown">
    Full control. Can share, change roles, and permanently delete the project. There is always exactly one owner.
  </Card>
  <Card title="Editor" icon="pencil">
    Can upload evidence, manage the project plan, run assessments, export, and share with others.
  </Card>
  <Card title="Viewer" icon="eye">
    Read-only access. Can view content and download exports.
  </Card>
</CardGroup>

## Where to manage it

Open the project you want to share and click **Share** in its header. From there you can invite a collaborator by email, change an existing collaborator's role, or remove access — including canceling a pending invitation for someone who hasn't signed up yet.

<Info>
  This is different from adding someone to your **workspace** — see [Manage workspace members](/settings/workspace-members) for team-wide collaboration versus per-project sharing.
</Info>

For the full step-by-step flow, permissions table, and API reference, see [Share and collaborate](/guides/share-and-collaborate).
