After uploading evidence, Nitrogen AI presents you with a set of module recommendations tailored to your project. Modules are the tools that generate your deliverables — investment memos, financial models, environmental plans, and more. You select which ones to run, supply any additional inputs they need, and for document modules you review and confirm an AI-generated outline before full generation begins.

## How module recommendations work

Nitrogen AI analyses the keywords in your project description and intake fields, then ranks modules by how well they match your project type. A mini-grid project with a grid-extension timeline will see the LCOE Model ranked highly; a project with a strong community component will see the Stakeholder Assessment near the top.

<Info>
  Recommendations are a starting point, not a constraint. You can select any module regardless of its ranking — and you can deselect recommended modules that aren't relevant to your work.
</Info>

## Select your modules

<Steps>
  <Step title="Open the module selection step">
    After uploading at least one evidence document, the chat displays the module selection widget. You'll see the full catalog with AI-recommended modules highlighted at the top.
  </Step>
  <Step title="Review the recommendations">
    Each recommended module shows a confidence indicator. Higher confidence means the module is a stronger match for your project description. Read the short description for any module you're unsure about.
  </Step>
  <Step title="Select the modules you want to run">
    Check the box next to each module you want to include. You can select multiple modules at once — they'll all be queued and can be generated together in a single pass.
  </Step>
  <Step title="Click Select tools">
    Click **Select tools** to confirm your selection and move to the inputs stage.
  </Step>
</Steps>

## Provide module inputs

Each module needs certain information to run. Most inputs — geography, target population, goal, budget range, timeline — are already captured from your project intake. If a module needs something that wasn't collected during intake, the chat will ask for it conversationally.

<Note>
  Input values are shared across modules. If two modules both need your project's installed capacity, you only enter it once. You can review and update all inputs at any time from the **Tool inputs** panel in your project.
</Note>

## Module types and what to expect

Different types of modules work differently after you select them. Here's what to expect for each category:

<CardGroup cols={2}>
  <Card title="Document modules" icon="file-lines">
    Investment Memo, Due Diligence Checklist, Stakeholder Assessment, Landscape Mapping, ESMP, and MEL Plan all go through an alignment step before generation. You'll review and confirm an outline first.
  </Card>
  <Card title="Quantitative models" icon="calculator">
    LCOE Model, Carbon Model, and Solar Estimate skip alignment entirely. Once inputs are complete, they open an interactive model widget in the chat where you can adjust parameters and recalculate.
  </Card>
  <Card title="Template Fill" icon="file-pen">
    Upload your own document template, and Nitrogen AI fills in the values from your project data. The output matches your template's structure and format.
  </Card>
</CardGroup>

## The alignment step (document modules)

For document modules, Nitrogen AI generates an alignment — a proposed outline — before writing the full deliverable. This is your opportunity to shape the structure of the document before any heavy generation work begins.

<Steps>
  <Step title="Review the proposed outline">
    The alignment appears in the chat as a structured list of sections with titles and brief descriptions of what each section will cover. Read through it carefully — this is exactly how the generated document will be structured.
  </Step>
  <Step title="Request changes if needed">
    If any section doesn't fit your project, type your feedback directly in the chat. For example:

    ```text
    Remove the carbon finance section — this project is not pursuing carbon credits.
    ```

    ```text
    Add a section on community ownership structure and revenue sharing.
    ```

    The AI will revise the alignment based on your input. You can iterate as many times as you need.
  </Step>
  <Step title="Confirm the alignment">
    When you're satisfied with the outline, click **Confirm**. This locks the structure and queues the module for full generation. The AI will use the confirmed outline, your project inputs, and your uploaded evidence to write every section.
  </Step>
</Steps>

<Warning>
  Once you confirm an alignment, the document structure is locked for that generation run. If you want a different structure, you can run the module again — each run creates a new instance with a fresh alignment.
</Warning>

## Interactive model widgets (quantitative modules)

Quantitative modules — LCOE, Carbon, and Solar — open directly as interactive model widgets in the chat. You can adjust inputs like CAPEX, capacity factor, discount rate, or system size and recalculate instantly without re-running the full module.

<Tip>
  Use the interactive widget to run sensitivity scenarios before you export. Adjust your WACC or CAPEX assumptions and see how the LCOE changes in real time before downloading the final spreadsheet.
</Tip>
