title: "Deliverables and exports in Nitrogen AI"
description: "Deliverables are the outputs generated by modules — documents and models you can export as DOCX or XLSX for investors, donors, or internal review."
---

A **deliverable** is the finished output produced when a module completes its generation run. Depending on the module, a deliverable might be a narrative document — such as an investment memo or an ESMP — or a structured financial model with editable inputs. Each deliverable is tied to the specific module instance that created it and can be exported to a file at any time.

## Deliverable types and export formats

The format a deliverable exports to depends on the type of output the module produces:

| Deliverable | Output type | Export format |
|---|---|---|
| Investment Memo | Narrative document | DOCX |
| Due Diligence Checklist | Structured checklist | XLSX |
| LCOE Model | Financial model | XLSX |
| Carbon Model | Emission reduction model | XLSX |
| Solar Estimate (PVWatts) | Energy yield model | XLSX |
| Template Fill | Filled document template | DOCX or XLSX |
| Stakeholder Assessment | Assessment document | DOCX |
| Landscape Mapping | Assessment document | DOCX |
| ESMP | Management plan | DOCX |
| MEL Plan | Monitoring plan | DOCX |

<Note>
DOCX files are formatted for direct use in Microsoft Word or Google Docs. XLSX files open in Excel or Google Sheets and preserve model inputs for further editing.
</Note>

## Versioning

Every time you run a module, a new deliverable version is created. The most recent completed version is displayed by default. Earlier versions from the same module are retained so you can compare outputs across runs.

## Accessing deliverables

You can access deliverables from two places within a project:

- **Project chat view** — completed deliverables appear inline in the conversation as interactive widgets, with options to view and export.
- **Editor panel** — a dedicated panel alongside the chat that shows the deliverable content in a structured view.

## Citations and traceability

When a module generates a narrative document, it logs the evidence sources it drew from. Citations reference specific chunks from the documents you uploaded as evidence, so you can trace every claim back to the source material.

## Exporting a deliverable

To export a deliverable, click the export button on the deliverable widget in the chat or editor panel. The platform fetches the content directly from the database and generates the file — no need to resend any data.

The export endpoint follows this structure:

```http
GET /api/v1/initiatives/{id}/deliverables/{tool_id}/export
```

The `tool_id` corresponds to the module that produced the deliverable (for example, `investment_memo`, `lcoe_model`, or `carbon_model`).

<Tip>
For financial models like LCOE and Carbon, make sure you have recalculated the model in the chat view before exporting. The export uses the most recent set of inputs stored in the database.
</Tip>
