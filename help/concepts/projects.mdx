A project in Nitrogen AI is your primary workspace for a sustainable development initiative. Everything you do — uploading evidence, generating a project plan, running assessments, and collaborating with teammates — happens inside a project.

## Creating a project

Projects are created conversationally, not through a form. Start a chat and describe what you're working on — a title is generated automatically, and Nitrogen AI infers whatever else it can (geography, sector, target population, goal) directly from your description. Nothing else is required up front, and every field can change later as the project evolves.

Projects vary widely, so there's no fixed intake schema: a clean-cooking pilot, a water utility upgrade, and a solar mini-grid will each surface different details from their descriptions, and that's expected. The one field the AI leans on most is the **description** itself — the richer it is, the better your project plan, assessment recommendations, and status assessment will be.

## Project URL

Each project has a stable URL generated from its title when it's created:

```text
/projects/my-solar-project
```

You can share this URL with any collaborator who has access to the project.

## Ownership and sharing

Projects can be kept private or shared with teammates. See [Share and collaborate](/guides/share-and-collaborate) for the full sharing flow, and [Workspaces](/concepts/workspaces) for team-wide collaboration that's separate from per-project sharing.

<CardGroup cols={2}>
  <Card title="Editor" icon="pencil">
    Can upload evidence, generate and edit the project plan, run assessments, and modify project details.
  </Card>
  <Card title="Viewer" icon="eye">
    Can read project content and download exports, but cannot make changes or run assessments.
  </Card>
</CardGroup>

<Info>
  The person who creates a project is its owner. Owners can share the project, change collaborator roles, and permanently delete the project.
</Info>

## Archiving and restoring projects

Projects support soft deletion — archiving moves a project out of your active list without permanently deleting it.

<Steps>
  <Step title="Archive a project">
    From the project menu, select **Move to trash**. The project is removed from your active list and placed in the archive.
  </Step>
  <Step title="Restore a project">
    Open your archive, find the project, and select **Restore**. The project and everything in it — evidence, project plan, assessment outputs, chat history — is fully reinstated.
  </Step>
  <Step title="Permanent deletion">
    From the archive, select **Delete permanently**. This also removes any stored files. This action cannot be undone.
  </Step>
</Steps>

## What lives inside a project

- **Project plan** — the AI-generated pillars and work items for the initiative
- **Evidence** — uploaded files the AI retrieves from and cites
- **Project materials** — reference files that are not indexed for retrieval
- **Assessment instances** — every run of an assessment, with its own workflow state and output
- **Project status** — the ongoing, AI-assessed health dashboard
- **Chat history** — the full conversation log for the project

<Tip>
  Keep one initiative per project. Mixing unrelated initiatives in a single project can cause the AI to draw on irrelevant evidence when generating outputs.
</Tip>

## Frequently asked questions

<AccordionGroup>
  <Accordion title="Can I rename a project after creating it?">
    Yes. You can edit the project title and other fields at any time from the project header. The project's URL slug is set at creation and does not change when you rename it.
  </Accordion>
  <Accordion title="What happens to assessment outputs if I remove a collaborator?">
    Content a collaborator generated remains in the project. Removing a collaborator revokes their future access but does not delete anything they created.
  </Accordion>
  <Accordion title="Can I move evidence between projects?">
    Not directly. Evidence and project materials are scoped to the project they were uploaded to. To reuse a document in a different project, upload it again there.
  </Accordion>
</AccordionGroup>
