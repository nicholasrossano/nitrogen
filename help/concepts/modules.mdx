Modules are the AI-powered tools that do the analytical work in Nitrogen AI. Each module takes your project data — intake fields, uploaded evidence, and any additional inputs it needs — and generates a specific deliverable, such as an investment memo, a due diligence checklist, or a financial model. You can run multiple modules on a single project, and each one produces its own versioned output.

## How modules are recommended

After you upload evidence and complete the intake stage, the platform recommends modules based on your project description. Recommendations are ranked by confidence — modules most relevant to your project type and keywords appear at the top. You can select any combination of modules regardless of the recommendation score.

## Inputs and the gather-inputs stage

Each module requires specific inputs to run. Some inputs — like geography, goal, and budget range — are collected during project intake. If a module needs information not yet captured, the platform asks for it through the chat interface during the "gather inputs" stage. You only need to provide each piece of information once; values are shared across all selected modules where applicable.

## Module instance status

When you run a module, the platform creates a **module instance** that tracks the generation job. Instances move through the following statuses:

| Status | Meaning |
|---|---|
| **Pending** | The instance has been created and is waiting to start |
| **Running** | Generation is in progress |
| **Complete** | The deliverable has been generated and is available |
| **Error** | Generation failed; you can retry from the project view |

You can run the same module multiple times on a project. Each run creates a separate instance with its own deliverable version.

## Alignment

Some modules — including the Investment Memo — support an **alignment** step before generation begins. Alignment lets you review and confirm the proposed outline for the document before the platform writes the full content.

<Tip>
Reviewing the alignment outline is the best opportunity to shape the structure of your deliverable. If a section is not relevant to your project, you can remove it before generation starts.
</Tip>

Once you confirm the alignment, the module proceeds to generation and produces the final deliverable.
